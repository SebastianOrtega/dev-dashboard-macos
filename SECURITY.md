# Security Policy

## Scope

This project is a local-only macOS tool for inspecting development processes and stopping them on the current machine.

## Reporting A Vulnerability

- If GitHub private vulnerability reporting is enabled for this repository, use it.
- If it is not enabled, avoid posting sensitive exploit details in a public issue.
- Open a minimal issue asking for a private contact channel if needed.

## What To Include

- A short description of the issue
- Affected version or commit
- Reproduction steps
- Expected impact

## Out Of Scope

- Issues that require root access or prior local compromise
- Reports against third-party tools like `lsof`, `ps`, Xcode, or Apple notarization itself
